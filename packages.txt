# System packages installed via apt-get on Streamlit Community Cloud.
#
# xgboost and lightgbm both link against libgomp (OpenMP). Most local
# dev machines already have it (via build-essential, conda, etc.), but
# Streamlit Cloud's minimal container does not — without this file,
# `import xgboost` / `import lightgbm` (triggered indirectly when
# unpickling models/best_model.pkl) fails with:
#
#   OSError: libgomp.so.1: cannot open shared object file:
#   No such file or directory
#
# This is one of the most common "works locally, breaks on Streamlit
# Cloud" issues for ML apps that ship a pickled XGBoost/LightGBM object.
libgomp1
